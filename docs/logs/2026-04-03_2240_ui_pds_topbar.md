# Optimizacion de Botonera PDS y Controles de Correo

**Fecha**: 2026-04-03
**Módulo afecado**: PDS (Pedidos de Servicio), Componente Topbar

## Qué se hizo
1. Se extendió `admin_layout.php` y `backoffice_user_banner.php` para soportar la variable inyectable `$topbarLeftHtml`.
2. Dentro de `CrmPedidosServicio\views\form.php`, se eliminó el espacio hueco del `.rxn-module-header` extrayendo las acciones (`.rxn-module-actions`) al buffer y enviándolas al bloque izquierdo del Banner. Esto compactó el formulario de PDS ahorrando totalmente una línea vertical de la UI.
3. Se integró lógica de chequeo de estado para el botón "Enviar por Correo". Si `$pedido['estado_ui']` no es "finalizado", el botón pasa a ser secondary y queda deshabilitado (`disabled`).
4. Se programó de forma dinámica el mensaje de confirmación del modal universal para que, si se envía el correo antes de impactar el cambio en Tango ERP, agregue al modal: "¡No te olvides de enviar el PDS a Tango después!".

## Por qué
El cliente solicitaba ahorrar espacio de encabezado y prevenir envíos de correos prematuros cuando el pedido aún está en estado Abierto y con duraciones parciales que podrían cambiar.
