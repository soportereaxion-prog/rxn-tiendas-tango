# [UX] - Confirmacion semaforo y feedback estandar

## Que se hizo
- Se agrego un modal liviano basado en Bootstrap para confirmaciones importantes.
- Se introdujo una paleta semaforo para feedback y errores: exito, advertencia, error e info.
- Se reemplazaron confirmaciones nativas en pedidos y articulos por el nuevo flujo visual.

## Por que
- Los dialogs nativos no acompañaban la estetica del sistema y resultaban pobres para acciones sensibles.

## Impacto
- Las acciones importantes ahora se confirman con una interfaz uniforme y mas clara.
- Los mensajes post-accion se ven con una identidad visual consistente y facil de leer.

## Decisiones tomadas
- Se mantuvo la solucion simple con Bootstrap y JS vanilla, sin agregar frameworks extras.
