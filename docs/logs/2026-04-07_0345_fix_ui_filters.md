# 1.2.6 - RXN Live: Corrección Visual de Filtros y Dropdowns

**Responsable:** Lumi / Charly
**Fecha:** 2026-04-07 03:40
**Entorno:** RXN Suite (CRM / Tiendas)

## 1. Qué se hizo
- **Sincronización DOM de Filtros Locales:** Se obligó a la arquitectura a regenerar toda la cabecera de la tabla (`renderPlana()`) durante la aplicación de la configuración almacenada (`applyViewConfig()`) para que los inputs recuperen visualmente sus valores guardados. (Antes funcionaba tras bambalinas, pero se perdían sus "value" en pantalla).
- **Indicador Visual de Motor BD Activo:** Se incorporó detección de parámetros de base de datos (`f[...][op]`) durante el armado de cabeceras, para que el ícono embudo de RXN Live se encienda en color azul/naranja validando a simple vista que el operador tiene un filtro pesado procesando detrás.
- **Recálculo de Posicionamiento Popper:** El submenú avanzado (que contiene Filtros locales + Filtros BD) se renderiza on-demand para evitar inflar el DOM. Esto causaba un problema de superposición en algunas resoluciones bajas. Ahora la interfaz gatilla un recálculo manual asíncrono (`window.dispatchEvent('resize')`) que posiciona el menú Dropdown evitando cortes o *clippings* al fondo del listado.

## 2. Por qué
Porque los inputs nativos locales (del DOM JS) quedaban vacíos visualmente post-reload, engañando al operador que consideraba que "no se había guardado", cuando en realidad la tabla sí filtraba. Además, el funnel gris no advertía el comportamiento oculto a nivel Backend. La mezcla causaba inseguridad productiva.

## 3. Impacto Operativo
- Interfaz WYSIWYG 100% sincera: Filtro puesto = texto visible.
- Menú dinámico que responde a todos los tamaños de grilla.
